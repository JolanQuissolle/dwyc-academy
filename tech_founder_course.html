Module 02

Intelligence Artificielle & LLMs

Comprendre techniquement ce qu'est un LLM, comment ça marche, et comment le builder peut l'utiliser pour créer des produits.

🔬
C'est quoi un LLM ?
Large Language Model — la base de tout

Un LLM est un modèle de deep learning entraîné sur des quantités massives de texte (billions de tokens) pour prédire le prochain token. En faisant ça à très grande échelle, le modèle développe des capacités "émergentes" : raisonnement, code, créativité.

L'architecture centrale s'appelle le Transformer — inventé par Google en 2017 dans le papier "Attention Is All You Need". Tout GPT, Claude, Gemini et Llama en sont des dérivés.

Les concepts clés à maîtriser

  • Token : unité de base du texte (~4 chars). "Bonjour" = 1-2 tokens. GPT-4 a une fenêtre de 128k tokens.
  • Context Window : quantité de texte que le modèle peut "voir" d'un coup. Plus c'est grand, plus il peut traiter de l'info.
  • Parameters : les "neurones" du modèle. GPT-4 ~1.7T params. Llama 3 70B = 70 milliards.
  • Training : Phase où le modèle apprend sur des données. Coûte des millions de $.
  • Fine-tuning : Adapter un modèle pré-entraîné à une tâche spécifique. Moins cher que le training complet.
  • RLHF : Reinforcement Learning from Human Feedback. Comment on aligne les modèles pour qu'ils soient utiles et sûrs.
  • Inference : L'utilisation du modèle pour générer des réponses. Coûte moins cher que le training.
  • Hallucination : Quand le modèle invente des faits. Un problème majeur et actif.

Prompt Engineering — parler aux modèles efficacement

# Mauvais prompt "Écris-moi un email de vente" # Bon prompt — avec rôle, contexte, format, contraintes """ Tu es un copywriter B2B spécialisé dans le SaaS. Contexte : Je vends un outil d'analytics à des CMOs de scale-ups françaises. Mon USP : +40% de ROI en 3 mois. Ton : professionnel mais direct, pas de jargon inutile. Écris un email cold outreach de max 150 mots. Format : Objet / Corps / CTA clair. """
Techniques de prompting à connaître

Chain of Thought (CoT) : Demander au modèle de "penser étape par étape" améliore drastiquement les performances sur les problèmes complexes. Few-shot : Donner des exemples dans le prompt. System prompt : Définir le comportement global du modèle avant la conversation. RAG : Injecter des données externes dans le contexte. Ces 4 techniques couvrent 90% des cas d'usage.

Module 03

Stack technique & Infrastructure

Tu n'as pas besoin de tout coder, mais tu dois comprendre comment une app est construite pour prendre de bonnes décisions avec tes ingénieurs.

Pour un founder non-tech

Tu n'as pas besoin de coder. Mais comprendre le stack te permet de : recruter des bons ingénieurs, évaluer la faisabilité d'une feature, comprendre les compromis, et ne pas te faire enfumer.

🏗️
Anatomie d'une application moderne
Front, Back, Database, Infra — les 4 couches

Frontend (ce que l'user voit)

  • React / Next.js : Le framework dominant. Fait par Meta (React) et Vercel (Next.js). 90% des startups l'utilisent.
  • Vue.js : Alternative plus simple à React. Populaire en Europe.
  • Tailwind CSS : Framework CSS ultra-populaire pour styler rapidement.
  • TypeScript : JavaScript + types statiques. Standard dans l'industrie.

Backend (la logique métier)

  • Node.js / Express : JavaScript côté serveur. Rapide à prototyper.
  • Python / FastAPI : Indispensable pour tout ce qui touche à l'IA/ML.
  • Go : Très performant pour les APIs à fort trafic.
  • GraphQL vs REST : Deux approches pour les APIs. REST = standard, GraphQL = flexibilité.

Bases de données

  • PostgreSQL : La BDD SQL de référence. Robuste, scalable.
  • MongoDB : NoSQL, documents JSON. Flexible mais attention à la structure.
  • Redis : Cache en mémoire. Pour les données fréquemment accédées.
  • Supabase / PlanetScale : DBaaS (Database as a Service). Postgres managé.

Infrastructure & DevOps

  • Docker : Containerisation. "Ça marche sur ma machine" résolu.
  • Kubernetes (K8s) : Orchestration de containers à grande échelle.
  • Vercel / Railway : Déploiement simplifié. Parfait pour les premiers mois.
  • Terraform : Infrastructure as Code. Décris ton infra en fichiers.

Business Models tech

Les modèles économiques qui marchent dans la tech, comment les VCs pensent, et les métriques qui comptent.

Les modèles de revenus tech

🔄
SaaS — Software as a Service
Abonnement mensuel/annuel. Le modèle roi de la tech. Revenue récurrent prévisible. Les VCs adorent. Ex: Slack ($20/user/mois), Salesforce, HubSpot.
MRR/ARRScalable
📏
Usage-Based / Pay-as-you-go
Tu paies ce que tu utilises. OpenAI charge par token, Stripe par transaction, AWS par compute. Alignement parfait valeur/prix. Complexe à modéliser.
Consommation
🏢
Enterprise / Licence annuelle
Grands contrats ($50k-$1M+/an). Cycles de vente longs (6-18 mois). POCs, security reviews, procurement. Mais stable et difficile à churner.
ACV élevé
🛒
Marketplace
Commission sur les transactions. AirBnb, Uber, Stripe. Problème du "cold start" (chicken-and-egg). Mais énormes effets de réseau quand ça marche.
HardNetwork Effects

Comment les VCs pensent — ce que tu dois savoir

📐
Fund Math
Un VC d'un fund de $100M a besoin d'un seul exit de $500M+ pour "retourner le fund". Donc ils cherchent des startups qui PEUVENT devenir des licornes. Ton TAM doit être >$1B.
🎲
Power Law
Dans un portfolio de 20 startups, 1-2 vont générer 90% des returns. Les VCs savent que la plupart vont échouer et investissent en conséquence. Pas une banque, une loterie.
🏋️
Moats (fossés compétitifs)
Ce qui protège ton business : Network Effects, Data Moat, Switching Costs, Brand, Patents, Distribution. Sans moat, tu es une commodité que quelqu'un va copier.
📈
Stades de financement
Pre-Seed ($0-2M) → Seed ($2-5M) → Series A ($5-20M) → B ($20-100M) → C+. Chaque stade a ses métriques attendues. Ne lève pas trop tôt.
Les top VCs à suivre (Twitter/X)

a16z (Andreessen Horowitz), Sequoia Capital, Y Combinator (YC), Benchmark, Accel, Index Ventures. Pour la France: Partech, Kima Ventures (Xavier Niel), Alven, Idinvest. Suis leurs blogs, leurs "memos" et leurs partners sur Twitter — c'est une formation continue gratuite.

Silicon Valley — Culture & Histoire

Comprendre la SV, c'est comprendre les règles non-écrites qui régissent tout le monde tech mondial.

Les cultures SV que tu dois comprendre

YC (Y Combinator) — La Mecque des startups

Y Combinator est le programme d'accélération le plus prestigieux au monde (accepte ~2% des candidatures). 3 mois à San Francisco, $500K pour ~7% de ta boite. Alumni: AirBnb, Dropbox, Stripe, Coinbase, Reddit. La "YC network" est le rolodex le plus puissant au monde.

Leur philosophie : "Make something people want". Fais quelque chose de difficile à faire mais facile à utiliser. Parle à tes users. Lance vite. Itère. Paul Graham's essays sont obligatoires.

Le "Move Fast and Break Things"

La devise originale de Facebook (maintenant renié). Culture de rapidité d'exécution vs perfection. La Silicon Valley valorise l'action sur la réflexion. "Done is better than perfect." L'itération rapide prime sur la planification.

Meritocracy vs Networking

La SV prêche la méritocratie — les meilleures idées gagnent. En réalité, le "warm intro" (introduction chaleureuse par quelqu'un en commun) est essentiel pour lever des fonds. Construire ton réseau n'est pas optionnel.

Founder OS — Comment penser et opérer

Les frameworks mentaux, ressources, et habitudes des meilleurs founders tech.

Les frameworks fondateurs à intégrer

🔺
Product-Market Fit
Le Saint Graal. Marc Andreessen : "PMF, c'est quand les clients achètent ton produit aussi vite que tu peux le produire." Avant PMF : tout est en feu. Après PMF : tout devient logique. Ne scale pas avant PMF.
Priorité #1
⚡
MVP — Minimum Viable Product
La version la plus simple qui teste ton hypothèse principale. Pas un produit à moitié fait — un produit complet mais très simple. Airbnb au début = un site WordPress + 3 photos. Stripe = 7 lignes de code.
Lean
🔄
Build — Measure — Learn
Le cycle de base du Lean Startup (Eric Ries). Construis une hypothèse. Mesure avec des vrais users. Apprends. Pivot ou persiste. Répète. La vitesse de ce cycle est ton avantage compétitif.
Lean Startup
🔍
Jobs To Be Done (JTBD)
Clayton Christensen. Les gens "embauchent" un produit pour faire un job. Milkshake example : McDonald's a augmenté ses ventes de milkshake en comprenant qu'ils étaient "embauchés" pour les trajets matinaux solitaires.
Clayton Christensen

Module 07

Glossaire tech complet

Les termes que tu vas entendre en réunion, en podcast, en levée de fonds. Comprends-les, utilise-les.

API
Application Programming Interface. Interface qui permet à deux logiciels de communiquer. OpenAI expose une API pour accéder à GPT-4. Stripe a une API pour les paiements. Sans APIs, l'internet moderne n'existe pas.
ARR / MRR
Annual/Monthly Recurring Revenue. La métrique santé d'un SaaS. ARR = MRR × 12. Les VCs se réfèrent à l'ARR pour évaluer la valeur d'une startup.
B2B / B2C
Business to Business vs Business to Consumer. Slack = B2B. Spotify = B2C. Les VCs préfèrent souvent le B2B (churn plus faible, ACV plus élevé). Le B2C est plus dur mais peut scaler plus vite.
Bootstrap
Créer une startup sans lever de fonds externes. Croître avec ses propres revenus. Indie Hackers promeut ce modèle. Alternatives : ramen profitable, default alive.
Cap Table
Capitalization Table. Tableau qui montre qui possède quoi dans la startup. Founders, employés (ESOP), investisseurs. Protège jalousement ta dilution.
Churn
% de clients/revenus perdus sur une période. Le killer silencieux des SaaS. Un churn de 3%/mois = tu perds 30% de tes clients par an. Négatif = expansions > churns = Saint Graal.
CTO / CPO / CFO
Chief Technical/Product/Financial Officer. Le trio de base d'une boite tech. CTO = code. CPO = vision produit. CFO = finances. En early stage, le CEO fait souvent tout ça.
Defensibility
Capacité à te défendre contre la concurrence. Un produit indefensible = copié en 3 mois. Les moats (network effects, data, switching costs, distribution) sont tes défenses.
DevOps / MLOps
Practices qui combinent développement et opérations pour un déploiement continu. MLOps = même chose mais pour les modèles de ML. Automatisation, CI/CD, monitoring.
FOMO
Fear Of Missing Out. Psychologie derrière les bulles d'investissement. Les VCs investissent parfois par peur de rater le prochain Airbnb, pas uniquement sur la logique.
GPU
Graphics Processing Unit. La puce qui fait tourner l'IA. Nvidia en fabrique 90% de ceux utilisés pour l'IA. Un H100 coûte $30,000. Avoir des GPUs = avoir du pétrole en 2024.
GTM
Go-To-Market. Ta stratégie pour lancer et distribuer ton produit. Top-down (grands comptes) vs bottom-up (product-led growth). Le GTM est souvent aussi important que le produit.
ICP
Ideal Customer Profile. Le profil précis de ton client parfait. "CTOs de startups Series A-B, 50-200 employés, dans la fintech, qui ont déjà un outil d'analytics." Plus c'est précis, mieux tu vends.
Open Source
Code source disponible publiquement. Linux, Python, React = OS. Llama de Meta = OS. Permet l'adoption rapide mais difficile à monétiser. Souvent : OS pour l'adoption + enterprise pour le revenu.
Pivot
Changement de direction stratégique basé sur ce qu'on a appris. Slack était un jeu vidéo. Instagram était Burbn (check-in). YouTube était un site de rencontres vidéo. Pivoter n'est pas échouer.
PLG
Product-Led Growth. Le produit lui-même est le moteur de croissance (vs sales-led). Slack, Figma, Notion = PLG. Les users viennent, utilisent gratuitement, convertissent eux-mêmes.
Runway
Combien de temps tu peux survivre avec ta trésorerie actuelle. Burn rate de $100k/mois + $1.2M en banque = 12 mois de runway. Toujours avoir 18 mois minimum.
SAFE
Simple Agreement for Future Equity. Document YC pour lever des fonds rapidement sans évaluer la boite. Convertit en equity lors du prochain tour. Standard pour pre-seed.
TAM / SAM / SOM
Total Addressable Market / Serviceable / Obtainable. Ton marché total, atteignable, et réaliste. Un VC veut voir un TAM >$1B. Ton SOM dans 3 ans = ta cible réelle.
Unicorn
Startup valorisée >$1 milliard. Était rare. Est devenu commun (~1200 dans le monde). Decacorn = >$10B. Hectocorn = >$100B (ByteDance, SpaceX). Le Saint Graal des VCs.
Vesting
Mécanisme qui donne les actions aux fondateurs/employés progressivement dans le temps. Standard : 4 ans avec cliff de 1 an. Protège la boite si quelqu'un part tôt.
Web3 / Blockchain
Internet décentralisé basé sur la blockchain. Bitcoin, Ethereum, NFTs, DeFi. La hype de 2021 s'est calmée mais la tech reste pertinente pour certains cas d'usage (finance, identité, ownership).
🚀 La suite de ton voyage

Ce cours est le point de départ, pas la destination. La tech évolue si vite qu'apprendre une fois ne suffit pas. Installe une routine : 30 min de veille par jour (Twitter/X + newsletters), 1 podcast/semaine, 1 article de fond/semaine. Dans 6 mois, tu ne te reconnaîtras plus.
