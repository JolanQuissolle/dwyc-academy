🧠 Le modèle mental — Dataset × Template

Le SEO programmatique repose sur un principe simple : une page = une requête = une réponse unique. Vous construisez un template une fois, vous le branchez sur une base de données, et chaque ligne génère une page SEO-optimisée automatiquement.

PAGES = Template(variables) × Nombre de lignes en DB
EXEMPLES :
"Meilleur [outil] pour [use-case] en [année]" × 500 outils × 20 use-cases = 10 000 pages
"[Ville] + [service]" × 300 villes × 15 services = 4 500 pages

🔍 Recherche de keywords — Trouver les patterns exploitables

01

Identifier les patterns à volume

Dans Ahrefs/Semrush, cherchez un mot-clé seed. Filtrez sur KD < 20 et Volume > 100. Exportez. Cherchez les patterns répétitifs : "[verbe] [objet] pour [persona]", "[comparaison] [A] vs [B]", etc.

02

Valider l'intent commercial

Googlez 10 keywords du pattern. Si les résultats top sont des articles de blog → contenu informatif. Si ce sont des pages produit/SaaS → intent transactionnel. Ciblez l'intent transactionnel en priorité.

03

Construire le dataset

Créez une table Airtable/Notion/DB avec toutes les variables du pattern. Enrichissez avec IA (descriptions, comparaisons, données spécifiques). Chaque colonne = une variable dans le template.

// 2. Metadata dynamique pour chaque page
export async function generateMetadata({ params }): Promise<Metadata> {
  const tool = await getToolBySlug(params.slug)

  return {
    title: `Les ${tool.alternatives_count} meilleures alternatives à ${tool.name} en 2025`,
    description: `Comparatif complet des alternatives à ${tool.name}. Prix, features, avis — tout pour choisir le bon outil pour ${tool.primary_use_case}.`,
    openGraph: {
      title: `${tool.name} alternatives — Comparatif 2025`,
      images: [{ url: `/api/og?tool=${tool.slug}` }]  // OG dynamique
    },
    alternates: {
      canonical: `https://monsite.com/alternatives/${tool.slug}-alternatives`
    }
  }
}

// 3. Le composant page
export default async function AlternativesPage({ params }) {
  const tool = await getToolBySlug(params.slug)
  if (!tool) notFound()

  const alternatives = await getAlternatives(tool.id)

  return (
    <main>
      {/* H1 optimisé avec mot-clé exact */}
      <h1>Les {alternatives.length} meilleures alternatives à {tool.name}</h1>
      {/* Schema.org JSON-LD */}
      <SchemaMarkup tool={tool} alternatives={alternatives} />
      {/* Tableau comparatif généré depuis les données */}
      <ComparisonTable items={alternatives} />
      {/* Internal links automatiques vers pages connexes */}
      <RelatedPages category={tool.category} />
    </main>
  )
}

📋 Schema.org — Structured Data

typescript — SchemaMarkup component
export function SchemaMarkup({ tool, alternatives }) {
  const schema = {
    "@context": "https://schema.org",
    "@type": "ItemList",
    "name": `Alternatives à ${tool.name}`,
    "numberOfItems": alternatives.length,
    "itemListElement": alternatives.map((alt, i) => ({
      "@type": "ListItem",
      "position": i + 1,
      "item": {
        "@type": "SoftwareApplication",
        "name": alt.name,
        "description": alt.description,
        "aggregateRating": {
          "@type": "AggregateRating",
          "ratingValue": alt.rating,
          "reviewCount": alt.review_count
        }
      }
    }))
  }

  return <script type="application/ld+json"
    dangerouslySetInnerHTML={{ __html: JSON.stringify(schema) }} />
}

🏗️ Architecture d'une machine cold email

01

Data Layer — Trouver et enrichir les leads

Apollo.io pour la recherche initiale (titre, entreprise, email). Clay pour l'enrichissement multi-sources : LinkedIn, site web, news récentes, levées de fonds, offres d'emploi. Chaque signal = un angle de personnalisation.

02

IA Layer — Personnalisation contextuelle

Clay envoie les données enrichies à un prompt IA (Claude/GPT) qui génère la ligne 1 personnalisée. Input : dernier post LinkedIn + description entreprise + titre du prospect. Output : première phrase qui prouve que vous avez fait vos devoirs.

03

Sending Layer — Deliverability en priorité

Instantly ou Smartlead pour l'envoi. Domaines secondaires warmés (3 semaines minimum), rotation des boîtes mail, limites d'envoi respectées (30-50/jour/boîte). Un seul domaine grillé = blacklist pour tous.

04

Optimization Layer — Mesure et itération

A/B test systématique : subject lines, call-to-action, longueur d'email. Cible : >40% open rate, >4% reply rate, <0.2% unsubscribe. En dessous de ces seuils, arrêtez et retravaill

📊 Stratégie d'Event Tracking — Taxonomie

La qualité de votre analytics dépend de votre taxonomie d'events. Une convention rigoureuse dès le départ évite des semaines de nettoyage.

typescript — Convention naming + tracking
// Convention : [objet]_[verbe] en snake_case
// Exemples : user_signed_up, project_created, payment_completed

// lib/analytics.ts — Wrapper centralisé
import posthog from 'posthog-js'

export const analytics = {
  // Identify l'user avec ses propriétés
  identify(userId: string, traits: Record<string, unknown>) {
    posthog.identify(userId, {
      email: traits.email,
      name: traits.name,
      plan: traits.plan,
      company: traits.company,
      created_at: traits.createdAt,
    })
  },

  // Track un event avec propriétés
  track(event: string, properties?: Record<string, unknown>) {
    posthog.capture(event, {
      ...properties,
      timestamp: new Date().toISOString(),
    })
  },

  // Events critiques typés (safety net)
  signUp(method: 'email' | 'google' | 'github') {
    this.track('user_signed_up', { method, source: getUtmSource() })
  },

  featureUsed(feature: string, context?: Record<string, unknown>) {
    this.track(`feature_${feature}_used`, context)
  }
}

// Cohorte 1 — Power Users (votre meilleur segment)
PowerUser = users who performed "core_action_performed" >= 5 times
             in the last 7 days

// Cohorte 2 — At Risk (churn probable)
AtRisk = users who did "session_started" last between 14-30 days ago
          AND had >= 3 sessions before that

// Cohorte 3 — Churned (confirmé)
Churned = users who did NOT perform any event in last 30 days
           AND were active in the 30 days before that

// Cohorte 4 — High Intent Free (candidats upgrade)

                  who hit a paywall event in last 7 days
                  AND have >= 10 core_actions total
