Qu'est-ce que Supabase ?

L'alternative open-source à Firebase — PostgreSQL + Auth + Storage + Realtime + Edge Functions dans une seule plateforme.

Architecture globale

Supabase est une couche d'abstraction au-dessus de PostgreSQL. Chaque projet Supabase est une instance Postgres dédiée, exposée via une API REST auto-générée (PostgREST), un websocket Realtime, un serveur Auth (GoTrue), et un serveur de stockage S3-compatible.

Pourquoi pas Firebase ?

  • SQL réel = requêtes complexes, joins, agrégations
  • Open-source = auto-hébergeable
  • PostgreSQL = écosystème massif (pgvector, PostGIS...)
  • Row Level Security = sécurité côté base de données
  • Pas de vendor lock-in

Custom JWT Claims (avancé)

Pour ajouter des claims custom (rôle, organisation...) dans le JWT, on utilise une PostgreSQL Hook Function :

sql
-- Fonction appelée à chaque émission de token
CREATE OR REPLACE FUNCTION public.custom_access_token_hook(event jsonb)
RETURNS jsonb
LANGUAGE plpgsql
AS $$
DECLARE
  claims jsonb;
  user_role TEXT;
BEGIN
  -- Récupérer le rôle depuis votre table
  SELECT role INTO user_role FROM public.user_profiles
    WHERE id = (event->>'user_id')::UUID;

  claims := event->'claims';
  claims := jsonb_set(claims, '{user_role}', to_jsonb(user_role));

  RETURN jsonb_set(event, '{claims}', claims);
END;
$$;
⚠️ Pattern à éviter : Ne stockez pas l'état auth dans votre propre state management. Utilisez onAuthStateChange comme source unique de vérité. Les sessions Supabase persistent dans localStorage automatiquement.

Fonctions stockées (RPC)

Quand PostgREST ne suffit pas, écrivez des fonctions SQL appelées via rpc(). Elles s'exécutent côté Postgres = performances maximales.

-- Exemple : incrémenter les vues atomiquement
CREATE OR REPLACE FUNCTION public.increment_views(post_id UUID)
RETURNS INT
LANGUAGE sql
SECURITY DEFINER  -- Exécute avec les droits du créateur
AS $$
  UPDATE posts
  SET views = views + 1
  WHERE id = post_id
  RETURNING views;
$$;

pgvector — Recherche sémantique

Extension PostgreSQL pour stocker et requêter des embeddings vectoriels. Idéal pour la recherche IA.

sql
-- Activer l'extension
CREATE EXTENSION IF NOT EXISTS vector;

-- Table avec colonne embedding
ALTER TABLE documents ADD COLUMN embedding vector(1536);

-- Recherche par similarité cosinus
SELECT id, title, 1 - (embedding <=> '[0.1,0.2,...]'::vector) AS similarity
FROM documents
ORDER BY embedding <=> '[0.1,0.2,...]'::vector
LIMIT 5;

-- Index HNSW pour les performances à grande échelle
CREATE INDEX ON documents USING hnsw (embedding vector_cosine_ops);
Module 03

Row Level Security

La sécurité au niveau de la base de données. Chaque ligne est protégée selon l'identité de l'utilisateur.

Sécurité critiquePoliciesauth.uid()RBACMulti-tenant

Concept fondamental

Sans RLS, n'importe qui avec votre clé anon peut lire toute la base. Avec RLS, PostgreSQL évalue une policy pour chaque ligne avant de la retourner. C'est transparent pour l'application — la sécurité est dans la base.

sql — pattern fondateur
-- 1. Activer RLS sur la table
ALTER TABLE posts ENABLE ROW LEVEL SECURITY;

-- 2. Sans policy = PERSONNE ne peut rien faire
-- 3. Créer des policies explicites

-- Tout le monde peut lire les posts publiés
CREATE POLICY "public read published" ON posts
  FOR SELECT USING (published = true);

-- Un user ne peut lire QUE ses propres brouillons
CREATE POLICY "owner read drafts" ON posts
  FOR SELECT USING (
    user_id = auth.uid()  -- auth.uid() = ID du user connecté
  );

-- Un user ne peut modifier QUE ses propres posts
CREATE POLICY "owner update" ON posts
  FOR UPDATE USING (user_id = auth.uid());

-- Insertion : forcer user_id = user connecté
CREATE POLICY "owner insert" ON posts
  FOR INSERT WITH CHECK (user_id = auth.uid());

USING vs WITH CHECK

ClauseOpérationRôle
USINGSELECT, UPDATE, DELETEFiltre les lignes existantes visibles
WITH CHECKINSERT, UPDATEValide les données écrites
⚠️ Pour un UPDATE, les deux clauses s'appliquent : USING détermine quelles lignes peuvent être modifiées, WITH CHECK valide les nouvelles valeurs.

Patterns avancés

RBAC — Rôles applicatifs
sql
-- Table des rôles
CREATE TABLE user_roles (
  user_id UUID REFERENCES auth.users,
  role    TEXT CHECK (role IN ('admin', 'editor', 'viewer'))
);

-- Policy admin : accès total
CREATE POLICY "admin full access" ON posts
  FOR ALL USING (
    EXISTS (
      SELECT 1 FROM user_roles
      WHERE user_id = auth.uid() AND role = 'admin'
    )
  );
Multi-tenant — Isolation par organisation
sql
-- Chaque ressource appartient à une org
CREATE TABLE documents (
  id      UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id  UUID NOT NULL,
  content TEXT
);

-- Table de membership
CREATE TABLE org_members (
  org_id  UUID,
  user_id UUID REFERENCES auth.users
);

-- Seuls les membres de l'org voient les documents
CREATE POLICY "org member access" ON documents
  FOR SELECT USING (
    org_id IN (
      SELECT org_id FROM org_members
      WHERE user_id = auth.uid()
    )
  );

postgres_changes — Écouter la DB

typescript
// Écouter les INSERT sur une table
const channel = supabase
  .channel('posts-changes')
  .on(
    'postgres_changes',
    {
      event: 'INSERT',   // 'INSERT' | 'UPDATE' | 'DELETE' | '*'
      schema: 'public',
      table: 'posts',
      filter: `user_id=eq.${userId}`  // Filtre optionnel
    },
    (payload) => {
      console.log('Nouveau post:', payload.new)
      // payload.old = ancienne valeur (UPDATE/DELETE)
      // payload.new = nouvelle valeur (INSERT/UPDATE)
    }
  )
  .subscribe((status) => {
    if (status === 'SUBSCRIBED') console.log('Connected!')
  })

// Désabonnement (IMPORTANT dans useEffect cleanup)
return () => { supabase.removeChannel(channel) }
⚠️ Activer Replication sur la table dans Dashboard → Database → Replication, sinon aucun événement ne sera reçu.

Policies Storage (RLS)

Les policies storage sont des policies SQL sur la table storage.objects. Elles permettent de contrôler qui peut lire, écrire, supprimer des fichiers.

sql
-- Permettre à un user de gérer SES propres fichiers
-- Le path doit commencer par son user_id
CREATE POLICY "user owns their files"
  ON storage.objects FOR ALL
  TO authenticated
  USING (
    bucket_id = 'avatars'
    AND (storage.foldername(name))[1] = auth.uid()::text
  );

-- Lecture publique d'un bucket public
CREATE POLICY "public read"
  ON storage.objects FOR SELECT
  USING (bucket_id = 'public-assets');

-- Helpers utiles
-- storage.foldername(name) → ['folder', 'subfolder']
-- storage.filename(name)   → 'file.jpg'
-- storage.extension(name)  → 'jpg'

Anatomie d'une Edge Function

typescript — supabase/functions/hello/index.ts
import { serve } from 'https://deno.land/std@0.168.0/http/server.ts'
import { createClient } from 'https://esm.sh/@supabase/supabase-js@2'

serve(async (req: Request) => {
  // CORS headers
  if (req.method === 'OPTIONS') {
    return new Response('ok', { headers: corsHeaders })
  }

  try {
    // Récupérer le JWT de l'user depuis Authorization header
    const authHeader = req.headers.get('Authorization')!
    
    // Créer un client Supabase avec le JWT de l'user
    // → les requêtes respecteront le RLS de cet user
    const supabase = createClient(
      Deno.env.get('SUPABASE_URL')!,
      Deno.env.get('SUPABASE_ANON_KEY')!,
      { global: { headers: { Authorization: authHeader } } }
    )

    const { data: { user } } = await supabase.auth.getUser()
    
    const { name } = await req.json()

    return new Response(
      JSON.stringify({ message: `Hello ${name}, user: ${user?.id}` }),
      { headers: { ...corsHeaders, 'Content-Type': 'application/json' } }
    )
  } catch (error) {
    return new Response(
      JSON.stringify({ error: error.message }),
      { status: 400, headers: corsHeaders }
    )
  }
})

const corsHeaders = {
  'Access-Control-Allow-Origin': '*',
  'Access-Control-Allow-Headers': 'authorization, content-type',
}

Cas d'usage — Webhook Stripe

typescript — webhook Stripe complet
import Stripe from 'https://esm.sh/stripe@12?target=deno'

const stripe = new Stripe(Deno.env.get('STRIPE_SECRET_KEY')!, {
  apiVersion: '2023-10-16',
  httpClient: Stripe.createFetchHttpClient()
})

serve(async (req) => {
  const sig = req.headers.get('stripe-signature')!
  const body = await req.text()

  // Vérifier la signature Stripe
  const event = await stripe.webhooks.constructEventAsync(
    body, sig, Deno.env.get('STRIPE_WEBHOOK_SECRET')!
  )

  // Client admin (bypass RLS pour les opérations système)
  const adminClient = createClient(
    Deno.env.get('SUPABASE_URL')!,
    Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!
  )

  switch (event.type) {
    case 'checkout.session.completed': {
      const session = event.data.object
      await adminClient
        .from('subscriptions')
        .upsert({
          user_id: session.metadata.userId,
          stripe_customer_id: session.customer,
          status: 'active'
        })
      break
    }
  }

  return new Response(JSON.stringify({ received: true }))
})
