Blocs & Pages

Tout dans Notion est un bloc. Maîtriser les blocs, c'est maîtriser le langage de base de l'outil.

La philosophie des blocs

Chaque élément que tu vois dans Notion — un paragraphe, une image, un titre, une todo — est un bloc. Tu peux déplacer n'importe quel bloc avec un drag & drop. Tu peux le transformer en un autre type. Tu peux le mettre en colonne. Cette flexibilité est ce qui rend Notion si puissant.

ℹ️
Une page est elle-même un bloc. Tu peux imbriquer des pages à l'infini. La structure de Notion = une arborescence de pages contenant des blocs contenant parfois d'autres pages.

Tous les types de blocs

Blocs de texte

📝 Text
Le bloc de base. Supporte le markdown : **gras**, *italique*, `code inline`, ~~barré~~, et les couleurs.
H1 / H2 / H3
Titres de 3 niveaux. #, ##, ### en markdown. H1 est le plus grand, génère une section navigable.
📋 Bulleted List
Liste à puces. - ou * pour créer. Tab pour indenter. Imbricable à l'infini.
🔢 Numbered List
Liste numérotée. Commence par 1.. La numérotation est automatique et se recalcule si tu réorganises.
☑️ To-do
Case à cocher. [] pour créer. Quand cochée, le texte est barré automatiquement. Parfait pour les checklists.
💬 Quote
Bloc de citation. " pour créer. Barre verticale à gauche. Pour mettre en avant une idée ou citer quelqu'un.
📢 Callout
Bloc coloré avec emoji. Très lisible pour les notes importantes, warnings ou tips. Personnalisable en couleur.
⁄ Divider
Ligne horizontale de séparation. --- en markdown. Pour structurer visuellement une page longue.

Blocs média & embed

🖼️ Image
Upload direct, URL externe, ou depuis Unsplash intégré. Peut être redimensionnée. Caption ajoutée dessous.
🎥 Video / Audio
Embed YouTube, Loom, Vimeo par URL. Player intégré directement dans la page. Pour les cours asynchrones.
📎 File
Attacher n'importe quel fichier (PDF, doc, zip). Stocké dans Notion. Lien de téléchargement automatique.
🔗 Bookmark
Aperçu enrichi d'un lien externe (titre, description, favicon). Plus joli qu'un simple lien texte.
💻 Code
Bloc de code avec coloration syntaxique. 30+ langages supportés. Bouton "Copy" automatique.
📐 Math (LaTeX)
Équations mathématiques en LaTeX. $$E = mc^2$$. Rendu en temps réel. Pour les ingénieurs et scientifiques.

Blocs de mise en page

📊 Table (simple)
Tableau simple (pas une database). Pour des données statiques non-filtrables. Différent d'une vraie base de données Notion.
📐 Columns
Divise la page en 2, 3, 4+ colonnes. Drag n'importe quel bloc vers le côté pour créer une colonne.
🔄 Synced Block
Bloc synchronisé entre plusieurs pages. Modifie-le à un endroit, il se met à jour partout. Idéal pour les footers et disclaimers.
🗂️ Toggle
Bloc accordéon. Masque du contenu jusqu'au clic. Parfait pour les FAQs, les détails techniques ou les longues notes.
🔑
Tip pro : Le bloc Synced Block est extrêmement sous-utilisé. Si tu as un "header" de navigation ou un disclaimer légal qui doit apparaître sur 20 pages, crée-le en Synced Block. Une seule modification se répercute partout.

Transformer un bloc

Clique sur les 6 points (⋮⋮) à gauche d'un bloc → "Turn into". Tu peux convertir un paragraphe en H1, une liste en toggle, etc. Très utile quand tu reorganises tes notes.

Structure recommandée d'un espace Notion

📁 Structure idéale pour une startup
🏠 Home # Page d'accueil avec liens rapides ├── 📋 Projects # Base de données projets ├── ✅ Tasks # Base de données tâches ├── 👥 CRM # Base de données contacts/clients ├── 📝 Docs # Wiki & documentation │ ├── 📖 Handbook │ ├── 🔧 Tech Docs │ └── 📊 Reporting ├── 💡 Ideas # Brain dump et idéation └── 🎯 OKRs # Objectifs et résultats

Bases de données

Le cœur de la puissance de Notion. Une database te permet de stocker, filtrer, trier et afficher des données structurées de multiples façons.

Database vs Table simple

Une table simple (bloc /table) est juste une grille statique, comme dans Word. Une database Notion est une vraie base de données : chaque ligne est une "page" avec des propriétés, elle peut être filtrée, triée, vue sous différentes formes, et reliée à d'autres databases.

💡
Pense à une database Notion comme à un tableur Excel intelligent où chaque ligne peut s'ouvrir comme une page avec du contenu riche (texte, images, notes), et où les colonnes peuvent être des dates, formules, relations vers d'autres tables, etc.

Les types de propriétés (colonnes)

📝 Title Le nom de l'entrée. Obligatoire. Unique par database. ex: "Lancement V2"
💬 Text Texte libre. Pour les descriptions courtes, URLs, notes. "En attente de design"
#️⃣ Number Nombre entier ou décimal. Supporte des formats : $, %, étoiles, barre. 1500 → $1,500
🔽 Select Choix unique parmi une liste d'options colorées. Parfait pour les statuts. 🟢 En cours
☑️ Multi-select Plusieurs choix possibles. Pour les tags, catégories. 🏷️ IA, Frontend
📅 Date Date et/ou heure, avec option de date de fin (pour les plages). Intégré au calendrier. 15 jan → 28 jan
👤 Person Membre de l'espace de travail. Pour assigner des tâches. @Alice
📎 Files & Media Upload de fichiers ou URL externe. Pour attacher des documents à une entrée. brief.pdf
✅ Checkbox Booléen vrai/faux. Utile comme flag, ou pour les formules conditionnelles. ✓ / ☐
🔗 URL Lien cliquable. Séparé du champ Text pour les URLs. notion.so/...
📧 Email / Phone Cliquable (ouvre le client mail / l'appel). Pour les CRMs. [email protected]
🔢 Formula Calcul basé sur d'autres propriétés. Le plus puissant. Voir module 06. if(done, "✅", "⏳")
↗️ Relation Lien vers une autre database. Crée des connexions entre tables. Voir module 05. → Projet: Feature X
📊 Rollup Agrège des données depuis une relation (somme, count, min, max). Voir module 05. Σ 5 tâches
🕐 Created / Edited time Automatique. Date de création ou de dernière modification. Non-éditable. 14 jan, 10:32
👁️ ID (unique) Identifiant unique auto-incrémenté. Parfait pour référencer des items (TASK-001). TASK-042

Créer une database — pratique

1
Créer une nouvelle database
Tape /table pour une database en vue tableau. Ou /board pour commencer en Kanban. Chaque database est aussi une page — elle peut vivre dans ta sidebar.
2
Ajouter des propriétés
Clique sur + à droite des colonnes existantes. Choisis le type. Renomme-la. Chaque propriété est une colonne visible dans la vue tableau, et un champ dans chaque entrée.
3
Configurer les options Select
Pour les propriétés Select/Multi-select, clique sur la propriété → "Edit property" → Ajoute tes options avec des couleurs. Exemple statuts : 🔵 Backlog 🟡 En cours 🟢 Terminé
4
Masquer des propriétés selon la vue
Dans la toolbar en haut → Properties → toggle off les propriétés que tu veux masquer dans cette vue. Elles restent dans la page, juste cachées dans cette vue spécifique.

Vues, Filtres & Tris

Une même database peut s'afficher de 6 façons différentes. Chaque vue peut avoir ses propres filtres et tris — sans affecter les données.

Les 6 vues disponibles

📋
Table (Tableau)
La vue classique type Excel. Toutes les propriétés en colonnes. Idéale pour voir beaucoup de données d'un coup. Tri par colonne d'un clic.
📌
Board (Kanban)
Cartes organisées en colonnes basées sur une propriété Select. La vue numéro 1 pour le project management. Drag & drop pour changer de statut.
📅
Calendar
Affiche les entrées sur un calendrier basé sur une propriété Date. Parfait pour les roadmaps, éditoriaux, ou plannings de sprint.
🖼️
Gallery
Cartes avec préview de l'image de couverture ou contenu. Idéale pour les portfolios, bases de connaissance visuelles, ou bibliothèques de ressources.
📜
List
Vue minimaliste. Juste le titre + quelques propriétés. Rapide à parcourir. Parfaite pour les notes, le journal, ou les listes de lecture.
📊
Timeline (Gantt)
Vue Gantt chronologique. Nécessite une propriété Date avec début ET fin. Parfaite pour les roadmaps produit et la gestion de projet.
💡
Power tip : Une database peut avoir autant de vues que tu veux, chacune avec ses filtres. Ex: une database "Tasks" avec une vue Board pour le Kanban quotidien, une vue Calendar pour la vue hebdo, et une vue Table pour l'export. Même données, 3 perspectives différentes.

Filtres — l'essentiel

Les filtres permettent de n'afficher que les entrées qui correspondent à certains critères, dans une vue spécifique.

🔍 Exemple : Vue "Mes tâches aujourd'hui"
# Filtres combinés avec AND Assigné à est Moi (current user) Statut n'est pas Terminé Date d'échéance est avant Demain # Résultat : Uniquement MES tâches non-terminées dues aujourd'hui
🔍 Exemple : Vue "Pipeline commercial" (OR logic)
# Filtres avec OR Statut est Prospect OU Statut est Proposition envoyée OU Statut est Négociation # Résultat : Toutes les opportunités "en cours" sauf Won/Lost

Tris — logique et priorités

Les tris s'appliquent dans l'ordre. Le premier tri est prioritaire, puis le second en cas d'égalité.

# Tri multi-niveau pour un project tracker 1. Priorité → Descendant # Haute priorité en premier 2. Date limite → Ascendant # Puis par deadline la plus proche 3. Assigné → Ascendant # Puis par ordre alphabétique du nom

Grouping — organiser par propriété

En vue Table ou List, le grouping permet de regrouper les entrées par valeur d'une propriété. Exemple : grouper par "Assigné" pour voir les tâches de chaque personne. Ou grouper par "Sprint" pour voir les sprints séparément.

⭐
Filtres dynamiques : Utilise le filtre "Me" (current user) dans la propriété Person — la vue affichera automatiquement les tâches de la personne qui est connectée. Chaque membre de l'équipe voit ses propres données dans la même vue.

Linked Database Views

Tu peux afficher une database existante sur une autre page, avec ses propres filtres. Exemple : sur ta "Home page", créer une vue filtrée de ta database Tasks qui ne montre que tes tâches du jour. Les données sont partagées, la vue est locale.

# Comment créer une Linked Database View 1. Sur ta page Home, tape /linked 2. Sélectionne ta database "Tasks" 3. La database apparaît avec ses données originales 4. Ajoute un filtre : Assigné = Moi + Statut ≠ Terminé 5. Cette vue est locale — elle n'affecte pas la database principale

Relations & Rollups

Le saut quantitatif de Notion. Relier des databases entre elles transforme une collection de tableaux en un vrai système de données connecté.

Les Relations

Une propriété Relation crée un lien bidirectionnel entre deux databases. Une tâche peut être liée à un projet. Un contact peut être lié à une opportunité CRM. C'est la base des systèmes complexes.

🔗 Exemple : Projects ↔ Tasks
## Database 1 : Projects │ Nom projet │ Statut │ Tasks (relation) │ │ Lancement V2 │ En cours │ → Task A, Task B │ │ Refonte UX │ Backlog │ → Task C │ ## Database 2 : Tasks │ Tâche │ Projet (relation) │ Assigné │ │ Task A │ → Lancement V2 │ Alice │ │ Task B │ → Lancement V2 │ Bob │ │ Task C │ → Refonte UX │ Alice │ # En cliquant sur "Lancement V2" → on voit Task A et Task B # En cliquant sur "Task A" → on voit son projet "Lancement V2"

Types de relations

🔄 Bidirectionnelle (défaut)
La relation est visible dans les deux databases. Si Task A est liée à "Lancement V2", dans le projet "Lancement V2" on voit aussi "Task A". Le plus courant.
➡️ Unidirectionnelle
La relation n'est visible que dans la database source. L'autre database n'a pas de propriété de retour. Utile pour éviter le bruit dans des databases très chargées.

Les Rollups

Un Rollup est une propriété qui agrège des données depuis une relation. Tu as une relation "Tasks" sur tes projets ? Avec un Rollup, tu peux calculer combien de tâches sont terminées, leur somme totale de points, leur date la plus tardive, etc.

Les fonctions Rollup disponibles

FonctionDescriptionExemple
Count allNombre total d'entrées liées5 tâches au total
Count valuesNombre de valeurs non-vides3 tâches avec deadline
Count uniqueValeurs uniques distinctes2 assignés différents
Count checkedNombre de checkbox cochées3 tâches terminées sur 5
SumSomme d'une propriété NumberTotal : 120 points
AverageMoyenne d'une propriété NumberMoy. : 24 pts/tâche
Min / MaxValeur minimum / maximumPlus proche deadline : 15 jan
Show originalAffiche les valeurs brutesAlice, Bob, Alice
Show uniqueValeurs sans doublonsAlice, Bob

📊 Exemple complet : Dashboard projet avec Rollups
## Database "Projects" avec Rollups depuis "Tasks" Nom │ Tasks │ Tâches totales │ Terminées │ % Done Lancement V2 │ [...] │ Count all → 8 │ Count ✓ → 5│ 62% Refonte UX │ [...] │ Count all → 3 │ Count ✓ → 0│ 0% # "% Done" = Formule : prop("Terminées") / prop("Tâches totales")

Système à 3 tables — exemple startup

Clients ←→ Projets ←→ Tâches ## Clients Startup ABC └── Projets liés : [Site web, App mobile] └── Rollup MRR : sum(Projets.Valeur) = $8,500 └── Rollup Tâches ouvertes : count non-done ## Projets Site web (Startup ABC) └── Client lié : → Startup ABC └── Tâches liées : [Design, Dev, QA] └── Rollup % avancement : percent checked ## Tâches Design Homepage └── Projet lié : → Site web └── Assigné : Alice └── Statut : ✅ Terminé
🧠
Le mindset "graph" : Pense tes données comme un graphe de nœuds connectés, pas comme des tableurs isolés. Clients → Projets → Tâches → Membres. Chaque relation est une arête du graphe. Les Rollups te permettent de remonter des informations à travers ce graphe.

Formules avancées

Les formules Notion te permettent de calculer, transformer et automatiser des valeurs dans tes databases. Une syntaxe proche d'Excel mais plus puissante.

⚠️
Les formules Notion 2.0 (sortie 2023) ont changé la syntaxe. On utilise maintenant prop("NomPropriété") au lieu de juste prop.NomPropriété. Ce cours utilise la nouvelle syntaxe.

Fonctions de base

Référencer une propriété

prop("Nom de la propriété") // Ex: afficher le texte d'une propriété "Titre" prop("Title")

Conditions — if()

if(condition, si_vrai, si_faux)
Si la condition est vraie, retourne la première valeur. Sinon, la seconde.
// Afficher un emoji selon le statut if(prop("Statut") == "Terminé", "✅", "⏳") // Priorité haute → badge rouge if(prop("Priorité") == "Haute", "🔴 Urgent", "🟢 Normal") // if imbriqués (multiple conditions) if(prop("Score") >= 90, "⭐ Excellent", if(prop("Score") >= 70, "👍 Bien", if(prop("Score") >= 50, "😐 Moyen", "❌ Insuffisant")))

Dates — calculs temporels

// Nombre de jours restants avant deadline dateBetween(prop("Deadline"), now(), "days") // Est-ce que la deadline est passée ? prop("Deadline") < now() // Afficher "En retard" si deadline dépassée et non terminé if( and(prop("Deadline") < now(), prop("Statut") != "Terminé"), "🔴 En retard", "🟢 OK" ) // Ajouter 7 jours à une date dateAdd(prop("Date de début"), 7, "days") // Formater une date formatDate(prop("Date"), "DD MMMM YYYY") // → "15 janvier 2025" // Extraire le mois, l'année month(prop("Date")) // → 1 (janvier) year(prop("Date")) // → 2025

Texte et chaînes

// Concaténer du texte "Bonjour, " + prop("Prénom") + " !" // Convertir en majuscules/minuscules toUpperCase(prop("Nom")) toLowerCase(prop("Email")) // Contient une chaîne contains(prop("Tags"), "IA") // Longueur d'un texte length(prop("Description")) // Remplacer dans un texte replaceAll(prop("Texte"), " ", "-") // "Hello World" → "Hello-World" // Convertir number en string format(prop("Prix")) + " €"

Maths

// Opérations de base prop("Revenus") - prop("Dépenses") prop("Tâches terminées") / prop("Total tâches") * 100 // Arrondir round(prop("Montant") * 1.2) // Arrondi au plus proche ceil(prop("Valeur")) // Arrondi au supérieur floor(prop("Valeur")) // Arrondi à l'inférieur // Min / Max entre deux valeurs max(prop("Valeur A"), prop("Valeur B")) // Valeur absolue abs(prop("Écart"))

Formules complexes — cas réels

Barre de progression visuelle

slice("██████████", 0, floor(prop("% Done") / 10)) + slice("░░░░░░░░░░", 0, 10 - floor(prop("% Done") / 10)) + " " + format(prop("% Done")) + "%"
Génère : ██████░░░░ 60% — une barre de progression visuelle dans une colonne texte.

Calcul de durée (jours ouvrés)

let( total, dateBetween(prop("Fin"), prop("Début"), "days"), semaines, floor(total / 7), total - (semaines * 2) )
Calcule les jours ouvrés (hors weekends) entre deux dates. La variable let() permet de déclarer des variables locales dans une formule.

Score de priorité pondéré

(prop("Impact") * 0.5) + (prop("Urgence") * 0.3) + (prop("Facilité") * 0.2)
Score RICE simplifié. Impact = valeur business (1-10), Urgence = timing (1-10), Facilité = facilité d'implémentation (1-10). Utilisé pour prioriser les features.

Badge de statut avec délai

if(prop("Statut") == "Terminé", "✅ Done", if( and( prop("Deadline") < now(), prop("Statut") != "Terminé" ), "🔴 En retard de " + format(abs(dateBetween(prop("Deadline"), now(), "days"))) + "j", "⏳ J-" + format(dateBetween(prop("Deadline"), now(), "days")) ) )
Affiche : ✅ Done | 🔴 En retard de 3j | ⏳ J-7 selon l'état de la tâche.
🧮
Toutes les fonctions Notion : La documentation officielle liste toutes les fonctions disponibles sur notion.so/fr-fr/help/formulas. Les plus utiles : if(), dateBetween(), formatDate(), contains(), let(), and(), or(), empty().

Automatisations

Les automatisations Notion native + les intégrations externes (Zapier, Make) pour que ton espace de travail tourne en arrière-plan.

Automatisations Notion native

Depuis 2023, Notion a intégré un moteur d'automatisation direct dans les databases. Accessible via le bouton "Automate" dans la toolbar.

Structure : Trigger → Condition → Action

TRIGGER (quand ?) ├── Propriété modifiée ├── Entrée ajoutée ├── Entrée supprimée └── Date atteinte (scheduled) CONDITION (si ?) # Optionnel ├── Propriété = valeur X ├── Propriété contient X └── Propriété est vide ACTIONS (faire quoi ?) ├── Modifier une propriété ├── Ajouter/retirer un tag ├── Assigner à quelqu'un ├── Envoyer une notification Slack ├── Créer une nouvelle page └── Ajouter à une autre database

Exemples concrets

📋 Auto-assign à la création
Trigger : Entrée ajoutée
Action : Assigner "Assigné" = current user
Toute nouvelle tâche est automatiquement assignée à la personne qui la crée.
📅 Notification deadline proche
Trigger : Propriété Date = aujourd'hui + 2 jours
Action : Notifier l'assigné + changer le statut en "Urgent"
🔄 Archivage automatique
Trigger : Statut = "Terminé"
Condition : Date modifiée > 7 jours

Zapier + Notion — Connexions externes

Zapier (et Make/Integromat) permettent de connecter Notion avec 5000+ applications. Un "Zap" = Trigger + Action entre deux apps.

Les Zaps Notion les plus utiles

1
Formulaire Typeform → Notion CRM
Quand quelqu'un remplit ton formulaire de contact → une nouvelle entrée est créée dans ta database CRM Notion avec toutes les infos. Zéro copier-coller.
2
Email → Notion Inbox
Chaque email reçu avec un sujet spécifique (ex: "NOTION:") est automatiquement créé comme entrée dans ta database "Inbox". Transforme ton email en système de tasks.
3
GitHub Issue → Notion Task
Quand une issue est créée sur GitHub → tâche créée dans Notion avec lien vers la PR. Les devs travaillent dans GitHub, le PM voit tout dans Notion.
4
Notion → Slack notification
Quand une propriété change dans Notion (ex: "Statut" → "Prêt pour review") → message automatique dans le channel Slack #product. Tout le monde est informé sans effort.
5
Stripe → Notion CRM
Nouveau paiement Stripe → mise à jour du client dans ton CRM Notion (MRR, date de souscription, plan). Ton CRM est toujours à jour automatiquement.
🔧
Make vs Zapier : Zapier est plus simple et user-friendly. Make (ex-Integromat) est plus puissant et moins cher pour les gros volumes. Pour débuter : Zapier. Pour des workflows complexes avec branches et transformations de données : Make.

API Notion & Intégrations

L'API Notion te permet de lire, créer et modifier des pages et databases programmatiquement depuis Python, JavaScript ou n'importe quel langage.

Setup — obtenir ta clé API

1
Créer une intégration
Va sur notion.so/my-integrations → "New integration" → Donne-lui un nom → Récupère le Internal Integration Token (ta clé API). Commence par ntn_.
2
Connecter l'intégration à une page
Va sur ta page/database Notion → "..." (3 points) → "Add connections" → Sélectionne ton intégration. L'API n'a accès qu'aux pages où elle est explicitement connectée.
3
Récupérer l'ID de la database
Ouvre ta database en pleine page → L'URL ressemble à notion.so/workspace/abc123def456.... La partie en gras (32 caractères) est l'ID de ta database.

Utilisation avec Python

pip install notion-client
import os from notion_client import Client # Initialisation notion = Client(auth="ntn_votre_cle_api") DATABASE_ID = "abc123def456..." # ─── LIRE une database ─── response = notion.databases.query( database_id=DATABASE_ID, filter={ "property": "Statut", "select": { "equals": "En cours" } }, sorts=[{ "property": "Deadline", "direction": "ascending" }] ) for page in response["results"]: title = page["properties"]["Titre"]["title"][0]["plain_text"] print(title) # ─── CRÉER une entrée ─── notion.pages.create( parent={ "database_id": DATABASE_ID }, properties={ "Titre": { "title": [{ "text": { "content": "Nouvelle tâche" } }] }, "Statut": { "select": { "name": "Backlog" } }, "Priorité": { "select": { "name": "Haute" } }, "Deadline": { "date": { "start": "2025-02-01" } } } ) # ─── MODIFIER une entrée ─── notion.pages.update( page_id="id-de-la-page", properties={ "Statut": { "select": { "name": "Terminé" } } } )

Utilisation avec JavaScript / Node.js

npm install @notionhq/client
const { Client } = require("@notionhq/client"); const notion = new Client({ auth: "ntn_votre_cle_api" }); const DATABASE_ID = "abc123..."; // Lire la database async function getTasks() { const response = await notion.databases.query({ database_id: DATABASE_ID, filter: { property: "Statut", select: { equals: "En cours" } } }); return response.results; } // Créer une entrée async function createTask(titre, statut) { await notion.pages.create({ parent: { database_id: DATABASE_ID }, properties: { Titre: { title: [{ text: { content: titre } }] }, Statut: { select: { name: statut } } } }); } // Appeler getTasks().then(tasks => console.log(tasks)); createTask("Déployer la V2", "Backlog");
